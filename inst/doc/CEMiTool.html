CEMiTool: Co-expression Modules Identification Tool

  • Basic usage

Basic usage

The CEMiTool R package provides users with an easy-to-use method to automatically implement WGCNA analyses with optimized parameters, gene set enrichment analysis and overrepresentation analysis.

In order to use CEMiTool, you only needs a data.frame containing you expression data, with the gene names in the rows and sample names in the columns, as following:

library("CEMiTool")

# Load expression data
data(expr)

As you include more informations to cemitool we can build a more complete object and as you will see more rich reports.

For instance, we can provide a sample annotation data.frame specifying a class for each sample.

# Load your sample annotation data
data(sample_annot)

ncores <- parallel::detectCores()
doParallel::registerDoParallel(cores=ncores)
# Reading GMT file
gmt_fname <- system.file("extdata", "pathways.gmt", package = "CEMiTool")
gmt_in <- read_gmt(gmt_fname)
# Reading interactions
int_fname <- system.file("extdata", "interactions.tsv", package = "CEMiTool")
int_df <- read.delim(int_fname)
# running cemitool
cem <- cemitool(expr, sample_annot, gmt_in, interactions=int_df, 
                filter=F, plot=T, split_modules=T)
